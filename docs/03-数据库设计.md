# 数据库设计

数据库：SQLite，由 Prisma 管理。Schema 定义见 [`prisma/schema.prisma`](../prisma/schema.prisma)。

## 1. 实体关系

```
Customer (1) ───< (N) VpsServer        一个客户有多台 VPS（VPS.customerId 可空，SetNull）
Customer (1) ───< (N) CustomerPayment  一个客户有多条收款记录（Cascade）
Provider (1) ───< (N) VpsServer        一个提供商被多台 VPS 引用（VPS.providerId 可空，SetNull）
VpsServer (1) ───< (N) VpsRenewal      一台 VPS 有多条续费记录（Cascade）
VpsServer (1) ───< (N) VpnNode         一台 VPS 有多个 VPN 节点（Cascade）
```

- 删除 VPS 时，其续费记录与节点级联删除。
- 删除客户时，其收款记录级联删除，名下 VPS 的 `customerId` 置空。
- 删除提供商时，引用它的 VPS 的 `providerId` 置空。

## 3. 财务汇总口径

收款已从 VPS/续费移到独立的 `CustomerPayment` 台账。汇总公式：

```
总成本(USD)   = Σ purchaseCostUsd + Σ VpsRenewal.costUsd + Σ VpsBalanceLog.topupUsd
总实付成本(CNY) = Σ purchasePaidCny + Σ VpsRenewal.paidCny + Σ VpsBalanceLog.paidCny
总收款(CNY)   = Σ CustomerPayment.amountCny
差额(CNY)     = 总收款(CNY) − 总实付成本(CNY)        // 正=盈余/客户多付，负=欠款
（自动续费充值的 paidCny 已计入总实付成本/结算）
```

- **全局**（`src/app/admin/page.tsx`）：对全部 VPS/续费/收款聚合。
- **按客户**（`src/app/admin/customers/[id]/page.tsx`、客户列表页）：仅对该客户名下的 VPS/续费/收款聚合。
- **客户公开页**（`src/app/view/[customerId]/page.tsx`）：展示该客户每台 VPS 的购买成本(USD)/实际付款(CNY)，以及合计「总购买成本(USD)」「总实际付款(CNY)」。
- 成本以 USD 记录、付款/收款以 CNY 记录；差额只用人民币口径，不做汇率换算。
