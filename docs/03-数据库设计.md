# 数据库设计

数据库：SQLite，由 Prisma 管理。Schema 定义见 [`prisma/schema.prisma`](../prisma/schema.prisma)。

## 1. 实体关系

```
Customer (1) ───< (N) VpsServer         一个客户有多台 VPS（VPS.customerId 可空，SetNull；新建 VPS 时必选）
Customer (1) ───< (N) CustomerPayment   一个客户有多条收款记录（Cascade）
Customer (1) ───< (N) CustomerRecharge  一个客户有多条充值记录（Cascade，客户级共享余额）
Provider (1) ───< (N) VpsServer         一个提供商被多台 VPS 引用（VPS.providerId 可空，SetNull）
VpsServer (1) ───< (N) VpsRenewal       一台 VPS 有多条续费记录（Cascade）
VpsServer (1) ───< (N) VpnNode          一台 VPS 有多个 VPN 节点（Cascade）
```

- 删除 VPS 时，其续费记录与节点级联删除。
- 删除客户时，其收款记录与充值记录级联删除，名下 VPS 的 `customerId` 置空。
- 删除提供商时，引用它的 VPS 的 `providerId` 置空。
- 自动续费 VPS 的余额改为**客户级共享**（`CustomerRecharge`）：名下所有 auto VPS 共用该客户余额；当前余额取最近一条充值的 `balanceAfter`。原 `VpsBalanceLog` / `VpsServer.balanceAmount` 已弃用（保留列避免线上重建表）。

## 2. 表结构

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | String(cuid) | ✓ | 主键 |
| `name` | String | ✓ | 名称，唯一 |
| `createdAt` | DateTime | ✓ | 创建时间 |

### 2.0b `Customer`（客户）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | String(cuid) | ✓ | 主键 |
| `name` | String | ✓ | 名称，唯一 |
| `note` | String? | | 备注 |
| `createdAt` / `updatedAt` | DateTime | ✓ | 时间戳 |

### 2.0c `CustomerPayment`（客户收款记录，独立台账）

| --- | --- | --- | --- |
| `id` | String(cuid) | ✓ | 主键 |

| `paymentProof` | String? | | 实际收款截图（`data/uploads/` 下文件名） |
| `note` | String? | | 备注（付款方式/覆盖哪些 VPS） |

| 字段 | 类型 | 必填 | 说明 |

| `customerId` | String | ✓ | 外键 → Customer，级联删除，建索引 |
| `amountUsd` | Float | ✓(默认0) | 本次充值（USD，计入总成本） |

| `rechargeDate` | DateTime | ✓(默认now) | 充值时间 |
| `paymentProof` | String? | | 付款截图（`data/uploads/` 下文件名） |

## 3. 财务汇总口径

收款已从 VPS/续费移到独立的 `CustomerPayment` 台账，充值移到 `CustomerRecharge` 台账。汇总公式：

```
总成本(USD)   = Σ purchaseCostUsd + Σ VpsRenewal.costUsd + Σ CustomerRecharge.amountUsd
总实付成本(CNY) = Σ purchasePaidCny + Σ VpsRenewal.paidCny + Σ CustomerRecharge.paidCny
总收款(CNY)   = Σ CustomerPayment.amountCny
差额(CNY)     = 总收款(CNY) − 总实付成本(CNY)        // 正=盈余/客户多付，负=欠款
（客户充值的 amountUsd 计入总成本、paidCny 计入总实付成本/结算）
```

- **全局**（`src/app/admin/page.tsx`）：对全部 VPS/续费/收款/充值聚合；并提供「按客户统计」表（每客户 总成本$/总实付¥/收款¥/差额¥）。
- **按客户**（`src/app/admin/customers/[id]/page.tsx`、客户列表页）：仅对该客户名下的 VPS/续费/收款/充值聚合；客户详情展示「当前共享余额」（最近一条 `balanceAfter`）。
- **客户公开页**（`src/app/view/[customerId]/page.tsx`）：展示该客户每台 VPS 的购买成本(USD)/实际付款(CNY)，合计「总购买成本(USD)」「总实际付款(CNY)」；含自动续费 VPS 时展示「当前充值余额(USD)」。
- 成本以 USD 记录、付款/收款以 CNY 记录；差额只用人民币口径，不做汇率换算。
