# 开发指南

## 1. 环境要求

- Node.js 20+
- npm

## 2. 本地启动

```bash
npm install
cp .env.example .env.local      # 修改 ADMIN_PASSWORD、SESSION_SECRET
npx prisma migrate dev          # 初始化数据库（生成 prisma/dev.db）
npm run seed                    # 可选：写入示例数据
npm run dev                     # http://localhost:3000
```

- 公开页：`/`
- 后台：`/admin`（首访跳 `/login`，输入 `ADMIN_PASSWORD`）

## 3. 环境变量

| 变量 | 用途 | 读取方 |
| --- | --- | --- |
| `DATABASE_URL` | SQLite 文件路径 | Prisma CLI 读根目录 `.env`；应用读 `.env.local` |
| `ADMIN_PASSWORD` | 后台登录密码 | 应用 |
| `SESSION_SECRET` | 会话签名密钥（建议 `openssl rand -hex 32`） | 应用 |
| `PORT` | 监听端口，默认 3000 | `npm run start` |

> 注意：Prisma CLI **只读根目录 `.env`**（仅需 `DATABASE_URL`）；应用运行密钥放 `.env.local`（开发）或 systemd/`.env.production`（生产）。两处的 `DATABASE_URL` 应保持一致。

## 4. 常用命令

| 命令 | 说明 |
| --- | --- |
| `npm run dev` | 开发服务器（热重载） |
| `npm run build` | 生产构建（含 `prisma generate`） |
| `npm run start` | 启动生产服务（读 `PORT`） |
| `npm run seed` | 写入示例数据 |
| `npx prisma migrate dev --name xxx` | 改 schema 后新建并应用迁移 |
| `npx prisma migrate deploy` | 生产环境应用迁移 |
| `npx prisma studio` | 可视化查看/编辑数据库 |
| `npm run test:e2e` | 运行 Playwright 端到端测试（无头） |
| `npm run test:e2e:headed` | 有头模式运行（可见浏览器操作） |
| `npm run test:e2e:ui` | Playwright UI 模式（可视化调试/单步） |

## 5.1 端到端测试（Playwright）

- 测试目录：`e2e/`，配置 `playwright.config.ts`（仅 Chromium，串行执行，复用已运行的 dev 服务，否则自动启动）。
- 用例：
  - `e2e/public-view.spec.ts`：`/view` 展示且不含金额/后台入口、无需登录、首页 `/` 未登录跳转登录、错误密码提示。
  - `e2e/admin.spec.ts`：登录 → 财务汇总 → 新建 VPS → 加节点 → 续费 → 删续费 → 复制 → 删除（自动清理数据）、退出登录后被拦截。
- 登录密码：测试默认用 `md@123456`，可用环境变量 `ADMIN_PASSWORD` 覆盖（见 `e2e/helpers.ts`）。
- 首次需安装浏览器内核：`npx playwright install chromium`。
- 注意：用例会读写本地 SQLite，创建的数据会在用例结束时删除；用唯一名称（时间戳后缀）避免冲突。
- **定位经验**：详情页有多个表单（续费/编辑）都含 `input[type="date"]`，定位时务必用 `page.locator("form", { hasText: ... })` 缩小范围；`waitForURL` 匹配详情页要排除 `/admin/vps/new` 自身。

## 5. 代码约定

- **服务端组件**默认（列表/详情页直接用 Prisma 读库渲染）；需要交互（表单、按钮）的拆成 `"use client"` 客户端组件，通过 `fetch` 调 API。
- **纯工具函数**（如 `toFormData`）放在非 client 模块（`vpsFormData.ts`），避免被服务端组件当作客户端引用导致 `is not a function`。
- 所有数据库访问走 `src/lib/db.ts` 的 Prisma 单例。
- API 入参统一用 `src/lib/validate.ts` 的工具归一化与校验。
- 日期/状态展示统一用 `src/lib/dates.ts`。

## 6. 二次开发示例

### 新增一个 VPS 字段（例如"机房 datacenter"）
1. 在 `prisma/schema.prisma` 的 `VpsServer` 加字段 `datacenter String?`。
2. `npx prisma migrate dev --name add_datacenter` 生成迁移。
3. `src/lib/validate.ts` 无需改（用现成 `optStr`）。
4. 在 `src/app/api/admin/vps/route.ts` 与 `[id]/route.ts` 的 `data` 里加 `datacenter: optStr(body.datacenter)`。
5. 在 `vpsFormData.ts`、`VpsForm.tsx` 加表单字段，按需在页面展示。

### 修改到期状态阈值
编辑 `src/lib/dates.ts` 的 `expiryStatus`（当前"即将到期"为 ≤7 天）。

## 7. 故障排查

| 现象 | 原因 / 处理 |
| --- | --- |
| `Environment variable not found: DATABASE_URL` | Prisma CLI 没读到根目录 `.env`，确认其中有 `DATABASE_URL` |
| `xxx is not a function`（服务端导入客户端模块） | 把纯函数移出 `"use client"` 模块（见第 5 节） |
| 后台一直跳登录 | Cookie 未带或 `SESSION_SECRET` 变更导致旧会话失效，重新登录 |
| 端口被占用 | `lsof -ti:3000 | xargs kill` 或改 `PORT` |
