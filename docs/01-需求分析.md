# 需求分析

## 1. 业务背景

为客户采购 VPS 服务器并在其上搭建 VPN 服务。当前缺少集中记录与展示工具，导致：

- 多台 VPS 的到期时间分散，难以及时续费，存在服务中断风险；
- 采购成本与客户收款没有统一台账，利润核算困难；
- 客户无法自助查看自己的服务器有效期与可用节点，沟通成本高。

本系统用于解决以上问题：**统一记录 VPS 资产与财务、管理 VPN 节点，并向客户提供一个无需登录的有效期展示页**。

## 2. 角色

| 角色 | 说明 | 权限 |
| --- | --- | --- |
| 管理员（运营者） | 采购 VPS、搭建 VPN、维护数据的人 | 登录后台，增删改 VPS / 续费 / 节点，查看财务 |
| 客户（访客） | 使用 VPN 服务的最终用户 | 无需登录，仅查看公开展示页（有效期、节点，**看不到金额**） |

## 3. 功能需求

### 3.1 VPS 资产管理
- 记录服务器**提供商**（如 Vultr / DigitalOcean / 搬瓦工）。
- 记录**基本配置**：CPU、内存、硬盘、带宽/流量、地区、IP 地址、操作系统。
- 记录**购买时间**与**到期时间**。
- 记录**运行状态**（运行中 / 已停用）。
- 支持新增、编辑、删除。
- **一键复制**：基于已有 VPS 复制配置快速新建，便于录入同一提供商的多台同配置 VPS（复制后名称加"-副本"、清空 IP，供修改后保存）。

### 3.1b 计费类型（固定期限 / 自动续费）
- **固定期限（term）**：一次性按 **月 / 季度 / 年** 购买，有完整到期时间，显示剩余天数，到期续费延长到期时间。新建/编辑时选择**购买周期**，到期时间按购买时间自动推算（仍可手动调整）。
- **自动续费（auto）**：无固定到期，按小时/月/季度/年根据账户余额自动扣费续费；展示蓝色「自动续费中」徽章，不显示倒计时；记录续费周期 + **周期费用(USD，仅展示)**。
- 自动续费**不再按单台维护余额**，改为**客户级共享充值余额**（见 3.4b）：详情页只读展示「所属客户共享余额」，充值在客户页面统一进行。
- **VPS 必须关联客户**（新建/编辑时「所属客户」为必填）。

### 3.2 续费（固定期限）
- 对固定期限 VPS 执行续费操作，填写续费后到期时间。
- 续费后系统将该 VPS 的到期时间更新为最新值。
- 保留每一次续费的历史记录（续费前后到期时间、金额、备注）。
- 校验：续费后到期时间必须晚于当前到期时间。
- 支持**删除续费记录**（仅删除历史记录，不改动 VPS 当前到期时间；如需调整到期时间在编辑 VPS 中修改）。

### 3.3 提供商与客户（单独维护）
- **提供商**：单独维护提供商列表（仅名称），新增/编辑 VPS 时从中选择，而非手填。
- **客户**：单独维护客户（名称 + 备注）；每台 VPS 归属一个客户。
- 删除提供商/客户时，被引用的 VPS 仅将对应字段置空，不删除 VPS。

### 3.4 财务记录与按客户结算
每台 VPS / 每次续费记录**两笔成本**（运营者支出）：
- **采购成本（USD）**：VPS 的美元标价成本。
- **实际付款（CNY）**：购买时实际花掉的人民币（含汇率/手续费差异）。
- **实际付款截图**：可上传付款截图（图片，存服务器本地），VPS 与每次续费各自一张；后台与客户专属公开页均可查看。

**客户收款不在 VPS 页面维护**，而是独立的**收款台账**（归属客户）：一笔付款可对应多台 VPS（统一付款），后续续费等可多次收款。

汇总（全局 / 按客户两个层级）：
- 总成本（USD）、总实付成本（CNY）、总收款（CNY）；
- **差额（CNY）= 总收款 − 总实付成本**（正=盈余/客户多付，负=欠款）。
- **概览页提供「按客户统计」表**：每个客户的 VPS 数、总成本$、总实付¥、收款¥、差额¥。

### 3.4b 客户充值（自动续费共享余额）
- 给客户的自动续费 VPS **提前充值**（预充值）：在客户管理页记录**充值美元(USD) + 实付人民币(CNY) + 充值后实际余额(USD) + 日期 + 截图 + 备注**。
- 名下所有自动续费 VPS **共享该客户余额**；**当前余额取最近一次充值的「充值后余额」**（只记充值不扣费）。
- **充值美元计入总成本、实付人民币计入总实付成本/差额**（与续费一致）。

### 3.5 VPN 节点管理
- 每台 VPS 下可记录多个 VPN 节点，节点**归属对应 VPS**。
- 节点信息：名称、协议（Shadowsocks / V2Ray / Trojan / WireGuard 等）、地址/域名、端口、配置（加密方式/密码等）、**订阅链接**、是否启用。
- 订阅链接在后台维护，客户专属公开页也展示（带复制按钮，方便客户导入）。
- 支持新增、编辑、删除。

### 3.6 客户专属公开页（无需登录）
**只按客户单独分享，不存在“展示所有客户”的全局公开页**（避免一次性暴露全部客户数据）。默认首页 `/` 与裸路径 `/view` 均指向/重定向到登录（需登录）。

**`/view/<客户ID>`**（在客户详情页、或客户列表「管理」旁的「复制 / 打开」按钮分享给对应客户）
- 仅展示该客户名下的 VPS，不含其他客户数据，无后台入口。
- 每台 VPS 展示：名称、提供商、配置、购买/到期时间、**剩余时间**（彩色徽章，仅一个）、**购买成本（USD）**、**实际付款（CNY）**、**付款截图**、已启用 VPN 节点。
- 顶部合计：**总购买成本（USD）**、**总实际付款（CNY）**（含续费与充值）；含自动续费 VPS 时展示**当前充值余额（USD）**。

### 3.7 管理后台（密码保护）
- 单管理员密码登录，顶部导航：VPS / 客户 / 提供商。
- VPS 列表（客户、提供商、IP、付费类型、创建时间、到期时间、状态、金额，斑马纹）+ 全局财务汇总卡片 + 按客户统计表。
- 提供商管理、客户管理（含每客户结算与收款台账）、VPS 增删改、续费、节点管理。

## 4. 非功能需求

| 类别 | 要求 |
| --- | --- |
| 鉴权 | 公开页只读无需登录；管理操作用单密码 + 签名 Cookie 会话保护 |
| 部署 | 可部署到 Ubuntu 24.04，**监听端口通过环境变量 `PORT` 配置** |
| 数据 | 使用 SQLite 单文件存储，便于备份迁移 |
| 易用性 | 中文界面；到期状态用颜色直观区分 |
| 安全 | 公开页对金额脱敏；密码不入库（存环境变量）；会话 Cookie 为 httpOnly |

## 5. 范围之外（本期未做）

- 多用户/多角色账号体系（当前仅单管理员密码）。
- 到期自动邮件/短信提醒（当前靠后台与公开页的颜色提示人工关注）。
- VPS 提供商 API 对接、自动开通/续费。
- 汇率自动换算（三笔金额由人工录入）。
