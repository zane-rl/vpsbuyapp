# 部署文档（Ubuntu 24.04）

目标：将应用部署到 Ubuntu 24.04 服务器，以 systemd 守护进程运行，监听端口通过环境变量配置。

## 1. 安装 Node 20

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v   # 确认 v20+
```

## 2. 部署代码与依赖

```bash
# 假设部署目录为 /opt/vpsbuyapp
sudo mkdir -p /opt/vpsbuyapp
# 将项目文件上传/克隆到该目录后：
cd /opt/vpsbuyapp
npm ci
```

## 3. 配置环境变量

供 Prisma CLI 使用（根目录 `.env`，仅需 DATABASE_URL）：
```bash
echo 'DATABASE_URL="file:./prod.db"' > /opt/vpsbuyapp/.env
```

应用运行密钥可二选一：
- **方式 A**：直接写进 systemd 单元的 `Environment=`（见第 5 节）。
- **方式 B**：新建 `/opt/vpsbuyapp/.env.production`，在单元里用 `EnvironmentFile=` 引用：
  ```
  PORT=3000
  DATABASE_URL=file:./prod.db
  ADMIN_PASSWORD=你的强密码
  SESSION_SECRET=用 openssl rand -hex 32 生成
  ```

生成会话密钥：
```bash
openssl rand -hex 32
```

## 4. 迁移数据库并构建

```bash
cd /opt/vpsbuyapp
npx prisma migrate deploy
npm run build
```

## 5. systemd 守护进程

模板见 [`deploy/vpsbuyapp.service`](../deploy/vpsbuyapp.service)。

```bash
sudo cp deploy/vpsbuyapp.service /etc/systemd/system/
sudo nano /etc/systemd/system/vpsbuyapp.service   # 修改 PORT/密码/密钥/路径/User
sudo systemctl daemon-reload
sudo systemctl enable --now vpsbuyapp
sudo systemctl status vpsbuyapp
journalctl -u vpsbuyapp -f                          # 查看日志
```

关键字段：
- `WorkingDirectory=/opt/vpsbuyapp`
- `Environment=PORT=3000`（**改这里即可改端口**）
- `ExecStart=/usr/bin/npm run start`
- `Restart=on-failure`（崩溃自动重启）、`WantedBy=multi-user.target`（开机自启）

> 确保 `WorkingDirectory` 及 SQLite 文件对 `User=`（如 www-data）可写。

## 6. 反向代理（可选，对外 80/443 + HTTPS）

模板见 [`deploy/nginx.conf.example`](../deploy/nginx.conf.example)。

```bash
sudo cp deploy/nginx.conf.example /etc/nginx/sites-available/vpsbuyapp
# 编辑 server_name 与 proxy_pass 端口，使其与 systemd 的 PORT 一致
sudo ln -s /etc/nginx/sites-available/vpsbuyapp /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# 申请 HTTPS 证书（自动改写 server 块）
sudo apt-get install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 7. 数据备份

数据分两部分，都需备份：
- SQLite 文件（`DATABASE_URL` 指向，如 `/opt/vpsbuyapp/prod.db`）。
- **上传的付款截图目录** `/opt/vpsbuyapp/data/uploads/`（需保证对运行用户可写、随项目持久化，更新代码时勿删除）。

```bash
cp /opt/vpsbuyapp/prod.db /backup/prod-$(date +%F).db
tar czf /backup/uploads-$(date +%F).tgz -C /opt/vpsbuyapp data/uploads
# 或加入 crontab 每日备份
```

## 8. 运维操作

| 操作 | 命令 |
| --- | --- |
| 改密码 | 编辑单元里 `ADMIN_PASSWORD` → `sudo systemctl restart vpsbuyapp` |
| 改端口 | 编辑 `PORT` → 重启服务（如有 Nginx 同步改 `proxy_pass`） |
| 更新代码 | 拉新代码 → `npm ci` → `npx prisma migrate deploy` → `npm run build` → `sudo systemctl restart vpsbuyapp` |
| 查看日志 | `journalctl -u vpsbuyapp -f` |
| 停止/启动 | `sudo systemctl stop/start vpsbuyapp` |

## 9. 上线检查清单

- [ ] `ADMIN_PASSWORD` 已改为强密码
- [ ] `SESSION_SECRET` 已用 `openssl rand -hex 32` 替换（勿用开发占位值）
- [ ] `DATABASE_URL` 在根 `.env` 与运行环境一致
- [ ] `npx prisma migrate deploy` 已执行
- [ ] `npm run build` 成功
- [ ] systemd 服务已 enable 并运行正常
- [ ] 已配置数据库定期备份
- [ ] （如对外）HTTPS 证书已配置
