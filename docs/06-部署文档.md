# 部署文档（Ubuntu 快速部署）

仓库：https://github.com/zane-rl/vpsbuyapp
适用：Ubuntu 20.04 / 22.04 / 24.04。目标：**几条命令部署启动，直接通过 IP:端口 访问**。

---

## 0. 一句话原理

应用是 Next.js 一体化服务（前后端 + API 同一进程），数据存 SQLite 单文件，付款截图存本地 `data/uploads/`。配好 `.env` → 装依赖 → 迁移 → 构建 → 启动即可。

> ⚠️ **顺序要点**：`npm ci` 的 postinstall 会执行 `prisma generate`，它依赖 `DATABASE_URL`。所以**必须先创建 `.env`，再 `npm ci`**。下面的脚本/命令已按此顺序。

---

## 1. 最快路径（一键脚本，推荐）

```bash
# 安装 Node 20 与 git（已装可跳过）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs git

# 拉代码
git clone https://github.com/zane-rl/vpsbuyapp.git
cd vpsbuyapp

# 一键：生成 .env（含随机会话密钥）→ 装依赖 → 迁移 → 构建
ADMIN_PASSWORD='换成你的强密码' PORT=3000 bash deploy/bootstrap.sh

# 用 pm2 守护启动（开机自启 + 崩溃重启）
sudo npm i -g pm2
pm2 start npm --name vpsbuyapp -- run start
pm2 save
pm2 startup    # 按它打印的命令复制执行一次

# 放行端口（如启用了 ufw 防火墙）
sudo ufw allow 3000/tcp || true
```

完成后访问：**`http://<服务器IP>:3000/login`**，用 `ADMIN_PASSWORD` 登录后台。
客户专属查看页在后台「客户 → 某客户 → 客户专属查看链接」复制分享。

## 2. 手动分步（等价于脚本，便于排查）

```bash
cd vpsbuyapp

# 2.1 生成 .env（Prisma 与应用共用此文件）
cat > .env <<EOF
DATABASE_URL="file:./prod.db"
ADMIN_PASSWORD="换成你的强密码"
SESSION_SECRET="$(openssl rand -hex 32)"
PORT=3000
EOF

# 2.2 安装依赖（会自动 prisma generate）
npm ci

# 2.3 初始化/升级数据库
npx prisma migrate deploy

# 2.4 生产构建
npm run build

# 2.5 启动（前台试运行）
npm run start
```

| 变量 | 说明 |
| --- | --- |
| `DATABASE_URL` | SQLite 路径，`file:./prod.db`（落在 `prisma/prod.db`） |
| `ADMIN_PASSWORD` | 后台登录密码（务必强密码） |
| `SESSION_SECRET` | 会话签名密钥，`openssl rand -hex 32` 生成 |
| `PORT` | 监听端口，默认 3000 |

> 可选：`npm run seed` 写入演示数据（客户/VPS/收款样例）。**生产环境通常不要执行**，它会清空并重置这几张表。

## 3. 进程守护：pm2 或 systemd（二选一）

### 3a. pm2（最快）
```bash
sudo npm i -g pm2
pm2 start npm --name vpsbuyapp -- run start
pm2 save && pm2 startup
# 常用：pm2 logs vpsbuyapp / pm2 restart vpsbuyapp / pm2 stop vpsbuyapp
```

### 3b. systemd
```bash
sudo cp deploy/vpsbuyapp.service /etc/systemd/system/
sudo nano /etc/systemd/system/vpsbuyapp.service   # 改 WorkingDirectory / PORT / User
sudo systemctl daemon-reload
sudo systemctl enable --now vpsbuyapp
sudo systemctl status vpsbuyapp
journalctl -u vpsbuyapp -f
```
> 用 systemd 时也可不写 `.env`，把变量写进单元的 `Environment=`；但 `prisma migrate deploy` 仍需要能读到 `DATABASE_URL`（保留根目录 `.env` 最简单）。

---

## 4. 绑定域名 + HTTPS（可选）

把外网 80/443 反代到本地 `PORT`：
```bash
sudo apt-get install -y nginx
sudo cp deploy/nginx.conf.example /etc/nginx/sites-available/vpsbuyapp
sudo nano /etc/nginx/sites-available/vpsbuyapp        # 改 server_name 和 proxy_pass 端口
sudo ln -s /etc/nginx/sites-available/vpsbuyapp /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# 免费证书（自动改写为 HTTPS）
sudo apt-get install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```
启用反代后，对外用域名访问，可把防火墙的 3000 端口关闭，仅放行 80/443。

```bash
cd vpsbuyapp
git pull

pm2 restart vpsbuyapp     # 或 sudo systemctl restart vpsbuyapp
```

## 6. 数据备份（两部分）

```bash
# 数据库
cp prisma/prod.db /backup/prod-$(date +%F).db
# 付款截图
tar czf /backup/uploads-$(date +%F).tgz -C . data/uploads
```
> `prisma/prod.db` 与 `data/uploads/` 需对运行用户**可写**、随项目持久化；更新代码（git pull）不会动它们。

## 7. 常见问题

| 现象 | 处理 |
| --- | --- |
| `npm ci` 报 `Environment variable not found: DATABASE_URL` | `.env` 未在 `npm ci` 之前创建。先建 `.env` 再重跑。 |
| 浏览器打不开 | 检查进程是否在跑（`pm2 ls` / `systemctl status`）、安全组/`ufw` 是否放行端口、是否用对了 `PORT`。 |
| 登录一直失败 | 确认 `.env` 的 `ADMIN_PASSWORD`；改完需 `pm2 restart` / `systemctl restart`。 |
| 端口被占用 | 改 `.env` 的 `PORT` 后重启；或 `sudo lsof -i:3000` 找占用进程。 |
| 截图上传后看不到 | 确认 `data/uploads/` 对运行用户可写。 |
