### 修复：HTTP 部署下登录后被打回登录页（会话 Cookie 丢失）
- 现象：HTTP 访问时密码正确返回 200，但 `/admin` 一直 307 回 `/login`。
- 根因：登录 Cookie 在生产恒为 `Secure`，而 Secure Cookie 仅在 HTTPS 下被浏览器保存/回传，HTTP 部署时被丢弃。
- 修复：`src/app/api/auth/login/route.ts` 的 `secure` 改为按请求协议判断（`req.nextUrl.protocol === "https:"` 或 `X-Forwarded-Proto: https`）。HTTP→不加 Secure（可用）；HTTPS/反代→加 Secure（受保护）。
- 验证：生产模式 HTTP 的 Set-Cookie 无 Secure、带 `X-Forwarded-Proto: https` 时有 Secure；登录后 `/admin` 返回 200；E2E 全绿。
- 文档同步：`06-部署文档.md` 常见问题表新增该条。
